Los cuatro que vamos a fijar

Clase, objeto, atributo, método

Todo con el mismo ejemplo: la clase Jugador que ya conocen.

el plano

Clase

Es el molde. Define qué atributos y qué métodos va a tener cualquier jugador, pero no es ningún jugador en particular. En UML es el rectángulo que ya dibujaron.

lo real

Objeto

Es una instancia concreta de la clase: un jugador con nombre propio, dorsal propio y goles propios. De una misma clase salen tantos objetos como necesiten.

el dato

Atributo

Es la información que guarda el objeto: nombre, dorsal, goles, cansancio. Cada objeto tiene su propia copia, con sus propios valores.

la acción

Método

Es lo que el objeto sabe hacer: marcarGol(), entrenar(). Un método casi siempre lee o modifica los atributos de ese mismo objeto.

La confusión más común

¿Esto es lo mismo que una tabla de base de datos?

Parecido, sí. Igual, no. Miren los mismos datos representados de las dos formas.

nombredorsalgolescansancio
Instanciá objetos en el laboratorio para verlos acá.

Los mismos datos, pero cada fila ahora "sabe hacer cosas": bajá al laboratorio y usá los botones de método de cada objeto.

La diferencia real: una fila de una tabla es solo dato quieto. Para hacerle algo (sumar un gol, bajar el cansancio) necesitás un programa aparte que lea la fila, la modifique y la vuelva a guardar. Un objeto, en cambio, junta el dato y el comportamiento en el mismo lugar: el objeto mismo tiene el método marcarGol(), no hace falta nada externo. Por eso una clase no es una tabla, y un objeto no es una fila: se parecen en la forma, pero un objeto además sabe actuar.

Para tener a mano

Resumen rápido

Clase

El molde. Se define una sola vez.

Objeto

Una instancia real de esa clase, con valores propios.

Atributo

Un dato que el objeto guarda (su estado).

Método

Una acción que el objeto puede hacer (su comportamiento).

Sistema de reservas para una peluquería

El problema

Una peluquería anota los turnos en papel o por WhatsApp, pero a veces se olvida de los horarios, se superponen reservas o no queda claro qué servicio pidió cada cliente. El negocio necesita un programa simple para:

  • Registrar clientes.
  • Registrar servicios.
  • Reservar horarios.
  • Ver qué cliente tiene turno.
  • Evitar horarios repetidos.

Todavía no vamos a programar nada. El objetivo es identificar clases, atributos y métodos, y que los métodos queden nombrados con buenas prácticas (verbos claros, que digan exactamente qué acción hacen).

Paso 1: tocá una casilla (por ejemplo "Cliente → Atributos"). Paso 2: tocá una etiqueta del banco para colocarla ahí. Cuando termines de ubicarlas todas, apretá "Evaluar".
